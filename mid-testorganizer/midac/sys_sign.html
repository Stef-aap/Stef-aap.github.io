Systeem signalen

Systeemsignalen reflecteren de inwendige toestanden van het programma, toetsen e.d. en kunnen gebruikt worden als ingangssignalen van rekenfuncties. Systeem signalen hebben in de verschillende modi van het programma (edit/record/review/export) soms een verschillend gedrag.

Systeem signalen zijn booleans, maar kunnen evengoed als ‘gewone’ ingangssignalen van rekenfunkties worden gebruikt. Bij gebruik in formules van de systeemsignalen geldt: TRUE=1, FALSE=0.

Systeemsignalen geven biezonder krachtige programmeer-mogelijkheden. Het is wel noodzakelijk dat men exact het gedrag van deze signalen kent, vandaar dat een aantal omschrijvingen soms nodeloos detailistisch lijkt.

1/2 Boolean_true / Boolean_false

Variabelen voor het testen van rekenfunkties, danwel gebruik als dummy parameter.

3 Record_mode

True als het programma bezig is met opnemen.

4 Opname_gestart

True als opname minstens éénmaal is gestart.

5 Sampling_gestart

True als sampling actief ???

6/7/8 Timer_trigger_knop / timer_enable_knop / timer_reset_knop

geen funktie meer (dus altijd false), in de toekomst beschikbaar als 3 vrij programmeerbare buttons.

9… 16 Marker_1 … Marker_8

True als de desbetreffende marker actief is.

  • Opname mode, true als desbetreffende functietoets ingedrukt, false zodra de marker naar bestand wordt geschreven
  • Review en Export mode, true als marker (gedurende 1 sample), anders false

17 Last_Sample

Deze variabele wordt in de export en review mode true op het moment dat het laatste sample wordt verwerkt.

Een typisch gebruik van deze variabele is het maken van een rapport op basis van een scan van de gehele opname. Tijdens de scan kunnen allerlei grootheden worden berekend, statische analyses worden uitgevoerd en dan … op het laaste sample 1 waarde rapporteren. Zie voor een concreet voorbeeld rekenfunktie 15, Autonome Neuropathie Analyse (30 minuten opname van diverse tests, 7 resulterende getallen als rapport-uitvoer).

18 First_Sample

Deze variabele is bedoeld om binnen een opname éénmalig een bepaalde actie te starten (bijvoorbeeld het resetten van een event-counter). De variabele wordt dan ook niet true bij een herstart na een pauze.

  • Opname-mode, true zodra de opname voor de eerste keer wordt gestart, false zodra eerste sample door een rekenkanaal is berekend (Timer1).
  • Review-mode, true als eerste sample wordt gedisplayed, false zodra eerste sample door een rekenkanaal is verwerkt.
  • Export-mode, true als eerste sample wordt ingelezen, false zodra eerste sample door een rekenkanaal is verwerkt

NB, als geen rekenkanaal wordt gebruikt, heeft dit signaal geen zin, dus resetten hoeft niet.

19 (Her-)Start

Deze variabele is bij iedere (her-)start van de opname true. Dit geldt in iedere programma-mode.

  

20 Some_windows_freezed 

Individuele vensters kunnen (tijdens opname?) stilgezet worden.

Allereerst moet dan in de vensterinstellingen van dat venster de checkbox Display Freeze worden aangekruist.

Ten tweede moet dan het systeem signaal Some_windows_freezed true worden gemaakt.

Voorlopig is een klein knopje beschikbaar op het hoofdform, naast de normale Display Freeze knop.

  

21 Clear_displays 

Door dit signaal true te maken (in het algemeen vanuit een rekenfunctie), kunnen displays worden gecleared en/of de xpositie van de signalen erin worden gereset. Zodra deze actie wordt uitgevoerd, wordt de valg zelf gereset.

In de individuele vensterinstellingen kan worden aangegeven wat een venster doet op deze clear actie. Voorlopig zijn hier de volgende mogelijkheden geïmplementeerd: niets, xpos=0, clear+xpos=0.
