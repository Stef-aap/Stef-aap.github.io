Data Backup

Bij het uitleveren van een programma aan een klant moeten er de volgende directories (inclusief subdirectories) worden opgeslagen op de BACKUP CDROM (als voorbeeld is MIDAC genomen)
...\...\ D5_midac \ *.*
... \ LIBS_D5 \ *.*            (zonder subdirectories)
daarnaast moet een kopie van de installatie CD worden bewaard (i.v.m. overkoepelende programma's als TestOrganizer bv)



Afhankelijk van de status van een module worden de sources van een programma opgeborgen in een beperkt aantal subdirectories (direct in de root van de data-disk)
DATA_ACTUEEL    (gebruik ik om programma's waaraan gewerkt wordt op te slaan)
DATA_DEFINITIEF   (gebruik ik om programma's waaraan niet meer gewerkt wordt op te slaan)
... OF .... iets anders herkenbaars
Voor programma's die bestaan uit meerdere executables, wordt een aparte directory aangemaakt, waarin alle executables + dll's worden opgeborgen. Vanuit deze directory kan het programma dus ook gerund worden. Tevens wordt deze directory gebruikt om installaties te creeeren en misschien zal er daarom nog wat meer in komen.
DATA_TO_EXE  
Sources van programma's worden opgeborgen in een subdirectory (van data_actueel, data_definitief of hoe ze ook heten mogen), die 2 elementen bevat: de versie van de Delphi-compiler en een herkenbare naam, zoals
D5_MIDAC
D5_PACEMAKER





Bibliotheken en modules die gebruikt worden door meerdere programma's worden opgeborgen in de root\LIBS_D5. Deze bevat momenteel 3 vast gedefiniëerde subdirectories:
LIBextra   voor het opbergen van algemene zelf geschreven routines
LIBextrn    voor het  opbergen van routines van anderen
LIB_TestOrganizer    voor het opbergen van routines die door meerdere modulen van de TestOrganizer worden gebruikt
Door  deze constructie behoeft bij geen enkel programma meer een library path te worden opgegeven, mits in Delphi, via Tools / Environement Options / Library al deze directories worden toegevoegd.


Bibliotheken

worden gebruikt om na een herinstallatie van snel alle componenten te kunnen installeren.

Deze worden opgeborgen in LIBS_D5.

Momenteel kennen we 1 bibliotheek MID_LIBS, welke waarschijnlijk bij ieder individu verschillend is. Misschien moeten we hier in de toekomst nog iets aan doen.
