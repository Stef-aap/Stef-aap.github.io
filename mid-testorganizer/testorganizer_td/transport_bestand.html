Transport Bestand 
Het transport bevat alle gegevens, noodzakelijk voor het opstarten van een test, de te ondernemen actie, de naam van de gewenste test, de naam van het module-instellingen bestand, het patientnummer en de gebruikte directories.
Daarnaast wordt het transport bestand gebruikt voor het terugmelden van de afsluitende resultaten van de meetsessie.

Er is gestreefd naar een uniforme opbouw van het transportbestand. Op deze wijze kan iedere module gebruik maken van 1 en dezelfde unit, welke het lezen en schrijven van het transport-bestand ondersteunt.
Voor enkele oude modulen worden specifieke settings gebruikt, zie:
 

Tranport Bestand Directories 
Hier worden de volgende paths opgegeven, inclusief de laatste backslash (als vereist). Voorbeeld

[Directories]
Patient Path=c:\data\~midorg\mid-data\patient\
Protocol Path=c:\data\~midorg\protocol\vraaglst\
Data Path=c:\data\~midorg\mid-data\vraaglst\
Help Path=c:\data\~midorg\d3_morg\htm-help\
Patient Path het path waar de relevante patient-bestanden (*.pat) staan
Protocol Path hier staan alle beschikbare protocollen voor de opgestartte module
Data Path datapath, speciaal bestemd voor de opgestartte module,
LET OP: het laatste subpath (i.h.a. de naam van het testprotocol) ontbreekt, hiervoor moet de opgestartte programma module zelf zorgen
Help Path het path waar de relevante helpbestanden voor de specifieke module staan
(voorlopig allemaal indentiek, dus gebruik geen voor de hand liggende namen als index.htm)

Start Parameters 
Hier staan de belangrijkste besturingscommando's voor de op te starten module. Voorbeeld:

[Start Parameters]
Patient Number=684
Module=Neuro Psycho
Action=Edit
Settings File=vrlst.vli
Protocol=test_all.txt
Suffix=0
Patient Number
Dit bevat de unieke patient-code, waarmee alle andere patientgegevens zijn op te halen. Hierdoor is ook het hoofdstuk Patient Gegevens geheel vervallen.

Module
Dit is slechts een tekstveld, waarmee de opgestartte module niets wezenlijks mag doen.

Action
De actie die een opgestartte module moet uitvoeren, mogelijkheden:

Edit    Record View    Rapport Export    Import Settings


Settings File
De bestandsnaam (inclusief volledig path, i.v.m. flexibiliteit) van het ini-bestand waarin alle globale instellingen van de module staan (bijvoorbeeld het default font voor de module). De inhoud van het module-instellingen bestand is geheel "eigendom" van de specifieke module en wordt hier daarom niet besproken.
De extensie van het module-instellingen bestand wordt vastgelegd per module, dat geeft iedere module de mogelijkheid een selectie te presenteren van alle beschikbare instellings-bestanden. Deze wordt voor de volledigheid echter wel meegegeven.

Protocol
Naam (inclusief extensie, exclusief path) van het gewenste protocol. Het opnemen van de extensie is bewust gedaan, om bij het upgraden van de programma's te kunnen overgaan naar een nieuwe extensie (tijdelijk zullen er dan 2 extensies bestaan).

Suffix
Het volgnummer van dit protocol bij deze patient (1..).
 

Programma Gegevens 
Hier staan wat algemene (onbelangrijke) gebruikers gegevens. Voorbeeld

[Programma Gegevens]
Titel 1=Academisch Ziekenhuis Nijmegen
Titel 2=afd. Algemeen Interne Geneeskunde
Patientgegevens 
Patientgegevens worden niet meer opgenomen, omdat de opgestartte module deze gemakkelijk zelf uit het patientbestand kan halen. Hierdoor wordt een grotere flexibiliteit naar de toekomst gegarandeerd.

Actie Edit 
Het opgegeven testprotocol wijzigen, of een nieuw testprotocol (als lege bestandsnaam) aanmaken.
    Uniformiteit:
Voor zover zinvol en mogelijk WYSIWYG editing.
Aan het eind van de edit-sessie opslaan via de standaard save-dialoog (welke zo volledig mogelijk is ingevuld).

Actie Opnemen 
Hiermee wordt de afname van de test gestart. Nadat de test is afgenomen, worden datum en time-stamp via het transport-bestand teruggegeven, zodat MIDORG de test kan afvinken.

Actie Bekijken 
Hiermee kan een reeds gemaakte opname worden bekeken.

Actie Export 
nog te definieren

Actie Import 
nog te definieren

Actie Rapport 
Voegt de resultaten van de gewenste opname toe aan ....
Als extra parameter wordt hier het gewenste rapport-nivo opgegeven.

Actie Instellingen 
Wijzigen van het opgegeven module instellingen bestand. Uiteraard bestaat de mogelijkheid bij het afsluiten het bestand onder een andere naam op te bergen.
Als de bestandsnaam leeg is, danwel niet bestaat, wordt het standaard module-instellingen bestand <application-naam>.ini gebruikt.
