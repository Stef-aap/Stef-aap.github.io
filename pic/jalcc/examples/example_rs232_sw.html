Introduction

This demo is exactly like the RS232-hardware demo, except instead of the hardware usart a pure software routine is used for the communication. This is especially meaningfull for debugging the smaller devices like the 12F675.

  

Again the demo program receives a character through the serial port, adds one to the value of that character and sends it back through the serial port.

Now you'll see that receiving is quiet difficult. Sending a single character will work perfectly ("1" and "2" in the example below). But sending a string line "123" will only return the first character correctly.

Still for debugging this is perfect, because you only want to send characters from the PIC.
