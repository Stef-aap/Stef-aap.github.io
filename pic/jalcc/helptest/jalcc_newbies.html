JALcc, first steps

  

  

Installation

  • Leave as much as possible "default" during the installation. 
  • At least install the program with the following points in mind 
  • short pathnames, without any points in it 
  • install close to the root of the harddrive 
  • Do not set any compatibility switches (if you don't know what this is: perfect !) 

  

  

First steps with JAL

  • Start the program and wait untill all the windows have opened 
  • press F5, this will compile the active program, and as an result you'll see an new window occuring, containing the assembler listing. Perfect: you compiled your first program correctly. If this doesn't work this way, look in the page about problems.
  • Now we're going to make anonther program active (you can leave all the windows open for the moment). On the left you see a white panel with 4 tabpages, go to the projectgroup page, abbreviated as "PG". Double click on the program e0002 (or click and press on open at the bottom). The tabs will jump back to the active project ("Proj"), the main jal file "e0002.jal" will be opened in the editor an you can see the hierachical structure of the program and it's relevant files. In this case all files (except the mainfile) are preceeded by an asterisk, indicating they are loaded from the general library path. 
  • press F5 again to compile the program. Now the compiler should give an error message and the editor should jump into the position where the error is, now try to correct the error . 
  • press F5 again, still another error is there, but now in one of the library files, which will be opened automatically. Correct this error ... Now you compiled your second program. 

  

  

Next steps with XCSB

  • download XCSB basic from the orginal site. 
  • JALcc has already made directories for the executables of XCSB:    ....\ JALcc \ XCSbasic \.... 
  • Go to the tab projectgroup and doubleclick test01.xcb. The XCSB project is opened, you get to see the file-hierarchy and the mainfile will be opened in the editor. 
  • press F5, in this case 2 errors should be reported and the cursor will jump into the first error. Correct the fisrt error. 
  • click in the errorlist on the second error and the file where the error is will be opened and cursor will jump into the error line. Correct the error and compile again. By now you should have compiled your first "doing nothing XCS basic" program ;-). 

At this point it would be a good idea to read the helppages, to get an idea of what is possible with this IDE.

On the other hand, the program has a good  windows-look-and-feel, so you could also use the trial and error method to find out.

Just one advise, for those who have no experience with projects and projectgroups, read at least the chapter "Projects".

  

  

My own programming with JAL

There's one testproject of my own in the projectgroup "stepper_test". It is a double 4-phase-stepper motor device, controled by an RS232 line.

WARNING: You you should only look at this example, after you've some experience with JAL.

You may notice some remarkable differences with the previous sampleprojects.

Remark how few lines the main program contains, combined with the fact that all the included files are preceeded by an asterisk (you should now by now what that means).

Now just look at the mainfile and click on the blue-book-button at the top, see what happens.
