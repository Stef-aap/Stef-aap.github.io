Why doesn't the program ask if it has to save a modified file ?

The program is based on minimal user interaction. In 99% of the cases most of the people always want to save their changes.

There are also a number of actions (e.g. compiling of code with macros), that needs to be saved before the real action takes place.

Another reason is that each modified document is saved at regular intervals.

 

If I change the read-only flag in a editor window, why isn't this reflected in the document list ?

There are several reasons for this behavior, first of all when a file is launched from the document list, the coupling between document list and editor is lost (I've seen document list with over 20,000 items (my own ;-), so it would take too much time to enumerate the whole document list. Secondly you could decide to make 2 branches with the same documents, one in read-only style, the other in edit-able style.


How do I create a new document ?

Select a document in the document (preferable in the directory where you want to put the new document), press the INS button (or select from the right mouse menu), and a new entry will be put in the document list. After opening the document, it will be created.

Especially with programs, you might follow a different way. Save an existing with "Save as", now the file will really be created and it will be put in the document list, just above the selected item.
