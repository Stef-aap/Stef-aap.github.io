MatLab - Python

 

         MainIndex



This document describes my experiences (with simple Scientific math), going from MatLab to Python. In this document it's assumed that always NumPy and SciPy are used, so we have the same basic computational elements. Although both programs look very similar, there are some small differences which are very important to notice.


General Feeling

This is a list of feelings (sometimes facts) about my first contact with Python, coming from MatLab. After writing it down in this list, an item it is never changed (it should represent the "first feeling"), but instead a remark is added when later on in time, the feeling/fact was incorrect.

  • Python has a much smaller footprint and starts much faster
  • The documentation and help-files are much better in Python. Most docs, cookbooks etc can easily be found on the web (without the very annoying MatLab log in !!)
  • The support in newsgroups for Python is much better, the people there, really try to help you instead of redirecting to whole bulk of data, in which the information is hidden. The traffic on the Python newsgroups is somewhat larger, then on the MatLab newsgroup. There's also no spam on the Python list, as there's much spam on the MatLab newsgroup. (SciLab newsgroup has much lower traffic.)
  • The Python books are much better, just compare the basic MatLab manual "Using MatLab, version 5" with the basic Phyton manual "Learning Python, O'Reilly"
  • MatLab is a scientific programming language only, Python is a general purpose programming language, which can do almost everything
  • Python is open source, MatLab is very expensive (outside an educational institute). A very important consequence is that you can traceback (and change) every detail in Python. Look in MatLab newsgroup how often someone wants to know (for some very legitimate reason) how a certain function performs it's calculation, and gets the official answer form Mathworks: "the only way to read and edit the built-in functions like FFT is to get a job with The MathWorks."
  • New versions of MatLab are full of bugs, Python is not.
  • Installation of MatLab is very simple, and looks very complex for Python. By using the "Enthought" edition, installing Python is a s simple as MatLab.
  • Uncertain what the far future for Python will be: Guido van Rossum has taken a job at Google end 2005, but he still is the "manager" of Python, Version 3000 announced, with good new features, but no backward compatibility (stepwise: depreciation warnings / error).
  • Python doesn't have the annoying whitespace in the command window (look at the screenshots below). For large arrays Python even has an option to only show the "corners" of a multi-dimensional array.
  • The "workspace" of MatLab is a great feature, it's global, easy to dig into the most complex variable. I miss that in my first steps in Python.
  • Module Import and Parameter passing to functions are very obscure in Python !!!! ???? But I'm beginning to see some light ...

> > You'd have to either pass the variables by reference into the function, [...]


In Python, all function calls are call by reference. Period. The key

difference between Python and other programming languages is in the

behavior of the assignment statement.


To paraphrase http://effbot.org/zone/python-objects.htm :


In C/C++, assignments modify objects by copying the value from the right

hand side into the memory allocated to the object on the left hand side.


In Python, assignments modify namespaces by making the name on the left

hand side become a reference to the object on the right hand side. It is

irrelevant whether the name referred to some other object before the

assignment, and it is irrelevant whether that object is mutable or not.


The only way to achieve call-by-reference-like behavior in the

"assignment modifies objects" sense is by passing a reference to a

mutable object and then invoking the object's methods to mutate it.

Maybe that's what you meant, but that wasn't clear from what you said.

Nobody mentioned the term for this. When a function changes its parameters or its environment, then it is called a "side effect" and generally it is considered harmful in all modern languages. ("Function" here refers to a piece of code that is used to calculate one or more values and provide these values to the caller.)


So you are trying to create a function that has a side effect. You should not do this, unless you have a really good reason.
