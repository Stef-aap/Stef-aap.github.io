JALss, history & downloads

 

 

Installation

Download the EXE-file in a temporary directory.

to install, just run the setup file (clean installation (except Bricxcc), files are only placed in the directory of your choice, no changes of the registry or whatsoever)

to uninstall, run unins000.exe, then delete the directory (to remove files that were changed by yourself)

Things that won't be changed (soon)

Scope running from top to bottom. Most people who ever used a scope like to see the scope running from left to right. Also I come up with this wish every month, but after a few days I return to the current view. The current method has a number of advantages:

  - the width of the computer screen is larger then the height (means more signals to view),

  - the drag and drop feature of the tabs is implemented in less then 20 code lines, implementing a rotated version would cost at least 400 program lines

  - unless we use the Chinese characterset, it's impossible to place the labels in a effective way

Version 0.3 , released ...

In the register watch window of a PIC, now also registers can be selected by SFR-name or by JAL-name. Not only more convenient to select, but also the thing you want for debugging. If a new compilation moves a variable to another place, the watch window still follows the correct variable.

In the scope, SFR's and JAL-variables are now stored by name. So JAL-vars will stay connected, even if they move due to a new compilation

When connecting a registers from a higher bank to the scope, programs reports an error (until infinity), bug fixed

8 bit registers now can also be shown as analog signals in the scope

Deleting a device can now also be done by DELETE-key (right-mouse menu remains possible)

There was a bug in deleting devices connected to the scope, fixed

Deleting a device, while simulation is running is now possible and simulation will start afterwards, if it was running before

Program hanged when a not existing register of a MCU-device was asked, now simulations stops, so the program can be stopped in a normal way.

Settings were activated after the program was restarted, now direct effectuation of changed settings

new device "byte LEDbar", connect with one click to a MCU-port or internal MCU register.LED's will show the value, which is also available as an analog output.

new devive "byte4-conversion", has 4 input bytes, which can be connected with just one click to a MCU-port or internal MCU register. The 4 (or less) bytes can be interpretated as one numerical value and the conversion can be selected.

when Java engine (and maybe some other programs) was running, sometimes IO-pins became white squares, bug fixed.

new device  "PIC16F873, 874 , 876 , 877 V0.2b"

device "PIC 16F627, 628  V0.2b", extended and improved
