LCD Display 

This device represents a LCD display based on the popular HD44780.

Not all features are implemented at the moment. The following things have to be done: 5*8 characterset instead of 5*7, only the first half of character ROM A00 is implemented (even with some differences), definitions of own characters (ASCII 0..7), shifting, cursor, display off.

The display can contain upto 80 characters, on 1 to 4 lines.

The HD44780 supports in fact a display of 1*80. Depending how the wiring to the chip is made in the LCD display, the start values of the individual lines (mapped into the 1*80 address space) can differ. In the code page you can set the start value of each line.

It only supports the 4-bit interface, because this is the mode with the least IO-pins.

Reading of RAM and ROM is not supported.

On reset the device even checks if the appropiate delay between the commands are according to the specifications, if not an error will be reported in the code list.

The code list gives the user an extra debug option. It lists all received bytes.

All numbers are displayed in a HexaDecimal presentation. 

A command is preceeded by "C=".

All commands and data bytes are translated to the full 8-bit commands / data,

except the first 4 reset bytes (3,3,3,2) which are given as a 4-bit presentation.

All buttons not mentioned are just for fun, try them.

 

The LCD component is based on the TIAEverIndicator of Igor Andronov, (thanks !)

7-Segment LED display  

The device can show from 1 upto 10  multiplexed 7-segment LED displays.

With the + or - button you can set for common anode (+) or common cathode (-).

With size, color and dot you can change the appearance of the display.

I find the colors awfull, so if anyone has good color ideas, I'll implement them.

The segments will light up, as soon as it's set on. But switching off is delayed to get a good view when multiplexing the displays. When simulation is stopped, the switch-off delay is set to zero, so you will see the exact result for that timepoint.

The device has build in current setting resistors, so you can simply connect ground and +5V to any segment.

LEDbar 

This is a very simple LEDbar, which can display from 1 upto 8 LEDs.

Momentary this device uses a reduced internal clock to refresh the LED-status, in the future this could possibly change into a permanent  "Slow-device".

     

   

  

Byte LED bar

This LED bar accepts a (named) 8-bit wide byte as an input. Therefor it can also be used to watch internal locations of a MCU. 

The name of the input byte is shown in the edit box. 

The input byte is also available as an analog output byte.

The input is sampled at 1 kHz.
