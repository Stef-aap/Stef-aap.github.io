JALss, Connections

  

  

 

Making connections is somewhat peculiar, due to the internal realisation of the connections (the internal structure is chozen for an optimal combination of speed and modularity).

The central point to start a new connection is always the input pin of an active device.

Let's first take a look at the different kind of IO-pins

  

 

Digital Input pin

The pin is dark green when it's not connected.

The pin is lime when it's connected to an output pin.

 

Digital Output pin

This pin is always Red

  

In principle only connections are valid, that connects an Input-pin to an Output-pin.

It's logical that an Input-pin can only be connected to one output pin. So trying to connect an input pin to a second output pin, will disconnect the input pin from the first output pin.

To connect / disconnect or reconnect an input pin, just click on it and a list of all available output pins in the circuit will be appear.

Some devices (like the scope) will show up a much larger list of available outputs, because it also can connect to internal pins/bytes/words of some devices.

Select the disered output pin from the list, double click on it (or press "select")  and the connection is there,. The input pin will now light lime and when slowly moving the cursor on to the input pin, a hint will popup, showing the name of the connected output pin and the device that owns that output pin.

An output can not be connected to any other pin. So clicking on an output pin will not generate any action.

In fact all analog pins are the same, that is they consist of ...

 

Analog Input pin

The pin is dark blue when it's not connected.

The pin is aqua when it's connected to an output pin.

 

Analog Output pin

This pin is always Yellow

  

Invisible Output pins

These will just show up in the output list  under static devices.

Examples are Ground, Vpos, Rpullup, Rpulldown
