Basic87 Terminal

  

Basic87 Terminal is a terminal program which is special adapted to BASIC87x from Myke Predko.

It's freeware and should run under win95, win98, winNT, win2000, winXP (winME is not garanteed).

Downloads

  

Basic87 is a very good starting point for beginners in the field of PICs. It gives you the possibility to learn the inside of PICs, without the knowledge of assembler and without the necessity to handle complex IDEs like MPLAB.

Writing larger programs is not easy with a normal terminal program, because you easy get lost in the single line editor. So I decided to write a simple editor, which can give you the complete overview of a program.

  

Basic87-Terminal consists of 3 windows and a number of  "speed-buttons".

The windows give you full control over your application, both by the basic terminal actions, but also by a full color-highligted editor (based on SynEdit).

The speedbuttons perform some of the basic tasks, without the need of knowing the command abbreviations.

Hardware Reset 

Performs a hardware reset of the basicPIC. 

(This requires that the MCLR-pin is connected to DTR of the RS232 port, see schematics)

  

  

 

Load from PIC 

Read the Basic program form the PIC and put it in the editor window.

(The current content of the editor window is lost)

Afterwards, all the variables are shown.

  

  

 

Download into PIC 

The contents of the editor window is downloaded into the basis-PIC.

Before downloading, the basic-PIC is erased.

Afterwards, all the variables are shown.

  

  

 

Get Variables 

Read all the variables and there value and display the in the variable window.

  

  

 

Download & Run 

The contents of the editor window is downloaded into the basis-PIC.

Before downloading, the basic-PIC is erased.

After downloading is completed, the current program in the basic-PIC is started.

Code completion

is default used to generate the register definitions.

For example you want to add the definition of TRISC, type a "T" or some more letters like "TRIS", then press ctrl-Spacebar.

Now a list will appear as shown below.

 

By selecting an item in the list and pressing Enter,

the definition of the register will be inserted, as shown below.

 

You can use the code completion for other purposes or add code strings, by editing 2 text files.

file:  "b87_itemlist.txt"

 

This files contains the list that will shown when pressing ctrl+Spacebar. It's best to place the items in the file in alphabetical order.

file:  "b87_insertlist.txt"

 

This file contains the strings that will be substituted, when the user selects an item. (the order must be the same as the other file)

  

  

Settings Commport

When the program is started the first time, or the selected commport is not available, the program will start with this window.

It's only possible to select available commports.

Normal Baudrate is 1200 Baud, but if you recompile the Basic87x sources higher baudrates must be possible.

In the modified hardware, DTR is used to initiate a hardware reset from the terminal program. If using a MAX232 interface chip, DTR should be inverted.

 

  

  

Editor Colors

Select an element from the list, and the actual colors (FG=foreground, BG=background) are shown in the color table.

By left-clicking on a color, the foreground color is changed, 

by right-clicking the background color of the selected element is changed.

The text attributes of each element can be set by the controls on the right.

With the FONT button, the overall font of the editor window can be changed.

The editor window at the bottom (in which you can really edit) is to demonstrate the color and attribute settings.

Commands, Reserved Words and Constants can be definied in the equally named tabs.

Preprocessor and Spave are not used.

Hardware FET

This circuit can be used to supply 10..30 mA from the serial port, so for simple experiments, no external power supply is needed.

On some labtops, this circuit will not deliver enough power.

Power can be improved by increasing C1 and C2 upto 1000 uF.

TxD and RxD are inverted, as with a normal MAX323. To use as little components, as necessary, DTR is not inverted. 

Terminal settings:

  DTR = inverted

  RTS  will be held high by program

BSP254A has the normal transistor connections:

Looking at the bottom, following the right arc:

Source - Gate - Drain

Emitter - Base - Collector

BSP254 has different connections !!

 

  

  

Hardware USB

Here a schematic with USBmod1 from Dontronics is given.

The advantage of this circuit is that you don't need an external power supply. The USB port can deliver upto 100 mA from the USB interface. (If you add an eprom to the USB-module you can even draw upto 500 mA from the USB port)

Terminal settings:

  DTR = NOT inverted
