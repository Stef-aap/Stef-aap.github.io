Device Overview

  

  

Introduction 

JALss is simple logic simulator (for M$-Windows), special designed to simulate PICs and other MCUs. Although it's basically designed for logical signals, it also has limited support for analog signals. JALss has special features from PIC programs written in JAL and is capable of communicating with JALcc (a JAL IDE)

JALss is freeware.

JALss has entry points to hook your own devices in the form of a DLL. A user device-DLL can be written in any language that supports the writing of windows-DLLs. 

The simulation is done with a 2-phase system clock. The frequency of this system clock is derived from the device with the highest frequency.

  • In the first phase all the devices read their input signals. 
  • In the second phase all the devices generate their output signals 

Calculation of the output signals can be done either in the first or the second phase, depending on where the calculation is done. So either the input signals or the output signals should be stored temporary depending on where the calculations are done.

 

  

  

  

Add Device

On the right the list of available devices is shown.

By selecting a device, followed by the ENTER key or the right-mouse button, a new instance of that device is created.

The organization of the list can be adapted by the user, through editing the file ....

see for more information ...

To each device, the user can make some small personal notes which are shown below the treelist.

With the blue splitter, the user can devide the space between the treelist and the notes.

 

  

  

Changing Device Settings

All devices hae a visual controls, to control the specific parameters of that device. 

All settings are automatically stored in the project file.

 

  

  

Add Connection

By clicking on a input pin of a device (the green pins for digital, the blue pins for analog), a form is shown with all available output signals.

SELECT: connects to the selected output

CUT: disconnects the input

CANCEL: doesn't change anything

If a device is selected (instead of a pin of a device, nothing will happen (same as Cancel).

The STATIC outputs are not from any visible device, but are just there.
