JALss, Building Devices

  

  

 

The simulation is done with a 2-phase system clock, which frequency is derived from the device with the highest frequency.

  • In the first phase all the devices read their input signals. 
  • In the second phase all the devices generate their output signals 

  

Calculation of the output signals can be done either in the first or the second phase. Depending on where the calculation is done, either the input signals or the output signals should be stored temporary.

Devices can be implemented in following ways:

  • integrated in JALss, can only be done by the writer / owner of this program 
  • as a simple DLL, making use of a standard available graphical form 
  • as a full DLL, with a completly self designed graphical form 

  

An DLL-device is a little slower than an integrated-device (about 10%) and (at the moment) lacks the special features of looking into the device.

When you build a device for the first time, it's advisable to build a simple-DLL-device first. When this simple-DLL-device works it's easy to translate it to a full-DLL-device.

  

  

Register the device

 After building a DLL-device, it must be "registrated", which is simply done by adding a few text lines to the ini-file.

Because every device has it's own unique ID-number, it's advisible to register the ID-number on this site, which garantees that future versions will not interfere with your own devices. And of course it would be wonderfull if you would make your devices available for other users.
