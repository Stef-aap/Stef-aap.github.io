JALss, File Data

  

  

 

DATA2FILE

This device, records up to 8 analog or digital signals into a file.

The column headers are derived from the signal names. For convenience you can override these names by your own names, by editing the fields behind the connection points. The form can be scaled to see the full signal names.

Only signals that are connected are recorded, so the number of columns will vary.

The filename can be editied or selected though the "New" button.

With the buttons Record / Stop, the recording can be started or stopped temporary.

With the value of Maxrec the maximum number of records (lines) that will be transported to disk can be set. If the maximum is reached, the recording is stopped, but the file is not closed. This gives the oppertunity to increase Maxrec to do another piece of recording. Reccount gives the actual recorded records.

 

  

  

When Time Column is checked, the first column in the export file will contain the time in seconds.

Pressing of New / Close / Stop / Record is quiet user friendly, the program will take all necessary actions if not the right order is used.

Once recording within a file is started, some of the parameters (signal-connections, signal-names, time-column) can not be changed anymore. Maxrec and clock-parameters can be changed during a recording.

Sampling can be done by an internal clock, of which the frequency can be set, or by an external clock, in which case the active edge can be selected.

The clock output is only there, when internal clock is selected and recording is active. In this way synchronisation of i.e. different files (both input and output) can be done.

The format of the file is a tab-delimited file, so it can be read automatically by all modern windows programs.

On the right, the result of a recording is shown in M$-Excel.

If you look at the graph carefully, you can see that the sample freqeuncy has changed during the recording.

 

  

  

Data output can be timelocked or synchronized with an internal or external clock. Timelock can only be selected if the datafile contains a time-column (in seconds) in its first column.
