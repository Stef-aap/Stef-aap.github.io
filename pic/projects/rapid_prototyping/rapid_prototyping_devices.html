Rapid Prototyping Devices

  

 

Rapid Prototyping Board

Details

   

rpd_all_sources.zip

With about 20 Rapid Prototyping Devices (RPD) you can build almost any functionality in just a few minutes. The board not only contains the normal RPD connectors, but also contains 4 Dwarf (TM from Voti) connectors, which extends the use of the RPD with all Dwarf devices.

If you're using JALcc, there's a "driver" for each RPD (and also for Dwarf), implemented as a 1 line macro. So you can concentrate directly on the real functionality of the software.

 

And finally (still just a dream) drawing of the final schematics, completely ready for lay-outing, should be possible with just 1 press on a key !!

USB Slave FT232RL

       


Details

This module was mainly build to evaluate the FT232R USB slave chip. But it can also be used as an alternative serial interface to the RPD or as an additional serial interface on the RPD.


To use it as the main serial interface of the RPD,

- remove the jumpers from the normal serial interface

- connect this device to J7

- place jumpers on both the analog and digital power

- place a jumper on the DTR-reset



PIC programmer


Details

The programmer is a simple, but fully complete, high-voltage programmer, capable of programming every PIC. The programmer can also be used for In Circuit Programming, through a 5-pin connector. The programmer software is divided in the PC-part and the PIC-part. To make it as flexible as possible for most users, the "intelligence" of the programmer is as much as possible put into the PIC. This makes it possible to add any programming algorithm, just by changing the code in the PIC. The PIC code is fully written in JAL.


The programmer should always be connected to J5 (because the program makes use of the PWM outputs)


 

Dwarf  DB009 / DB010

The Dwarf Module contains 8 small (DB009) or 8 large (DB010) LEDs.



Dwarf boards are a trademark of and available from Voti.

JALcc has a macro, which makes the LEDs available either as a whole byte or as individual LEDs. The macro also sets the direction bit correctly.



The macro compensates for the irregularities in the A-port (combined with the the E-port). Specifying the C-port, will leave RC6 and RC7 free for serial communication. By specifying "C+" instead of "C", you can force the macro to also include the RC6 and RC7 bits. Below the macro expansion for the (normal) portB is shown.

 


 

Mini Rapid Prototyping Board

Details

   

rpd_all_sources.zip

The Mini Rapid Prototyping Board, is meant to control 1 RPD device with a 12F675. De module is mounted on a standard programmer, which also must supply the power.

The mini-RPD can also be used as a program adapter, to program the 12F675 from a programmer with a 28-pins 16F877 foot. There's one disadvantage, the RPD-device must be disconnected during programming.

In the future it might be better to mount 2 ML-10 connectors, so also 2 small RPD devices can be used.
